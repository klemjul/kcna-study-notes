# Cloud Native Architecture

## Core concepts and definitions

- **Cloud native computing** uses an open source software stack to deploy applications as microservices, package each part into its own container, and dynamically orchestrate those containers to optimize resource use.
- A **microservices** approach splits a system into smaller services with clearer boundaries, independent deployment, and network-based communication.
- **Immutable infrastructure** means replacing deployed units instead of changing them in place, which improves consistency and repeatability.

### Cloud native ecosystem and principles

- Cloud native systems emphasize **automation**, **declarative configuration**, **resilience**, **observability**, and **scalability**.
- CNCF hosts widely used cloud native projects and provides a neutral place for collaboration across vendors, maintainers, and end users.
- Kubernetes is one part of the ecosystem; observability, networking, runtime, packaging, and security tools also matter.

### Cloud native community and collaboration

- The **Cloud Native Computing Foundation (CNCF)** is part of **The Linux Foundation**.
- Open source cloud native projects grow through shared standards, public documentation, maintainers, contributors, and community collaboration.
- For KCNA, know the role of CNCF as a foundation and ecosystem home, not just as a list of tools.

## Key commands and examples

- There are no must-memorize commands unique to this domain, but you should recognize common project categories such as runtime, observability, service mesh, packaging, and platform tooling.

## Exam-focused reminders

- Cloud native is broader than Kubernetes alone.
- Focus on **principles and ecosystem roles**, not deep product configuration.
- Be ready to distinguish a **foundation**, a **project**, a **principle**, and an **implementation tool**.
